Expand description

Raw manifest as it appears on disk or on the wire. Deserializes from TOML/JSON directly. May contain field combinations that violate cross-field invariants — call UncheckedManifest::validate to obtain a verified PackageManifest.

§Example

use tau_domain::UncheckedManifest;
// toml::from_str::<UncheckedManifest>(&raw)?.validate()?;

§plugin: Option<PluginManifest>

Plugin manifest declared via the [plugin] table.

None for data-only packages (no plugin table). Some for plugin packages — tau-pkg uses this to gate the build step during install (see plugin-loading spec §6.1, §6.3).

§sandbox: PluginSandboxRequirements

Plugin-side sandbox requirements declared via [sandbox] table.

Optional. Default = PluginSandboxRequirements::default() (no tier floor; auto-derived shapes). See PluginSandboxRequirements.

§skill: Option<SkillManifest>

Skill manifest declared via the [skill] table.

None for non-skill packages (no skill table). Some for skill packages — tau-pkg::skill_check (Skills-2) uses this to gate SKILL.md validation during install. See ROADMAP §16 and docs/decisions/0025-skills-foundation.md.

pub fn validate(self) -> Result<PackageManifest, PackageManifestError>

Run cross-field validation. Returns the validated manifest on success.

Field types are already validated at construction (PackageName, PackageSource, etc.); this checks invariants those types can’t enforce alone (non-empty description, non-empty Custom capability names, etc.).
