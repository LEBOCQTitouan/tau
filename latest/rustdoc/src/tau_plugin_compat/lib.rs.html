1//! Plugin compatibility verification harness — sub-project B (2026-05-04).
2//!
3//! This crate is test infrastructure only (`publish = false`). It exists
4//! because Task 1 of sub-project B established that all 5 real shipped
5//! plugins (anthropic, ollama, openai, fs-read, shell) declare
6//! `[sandbox] required_tier = "strict"`, and we need automated tests that
7//! verify each plugin actually works end-to-end under real sandbox
8//! enforcement (not just `MockSandbox`-based theory).
9//!
10//! # Layout
11//!
12//! - `fixtures/controlled-env-binary/` — standalone Cargo project (NOT a
13//!   workspace member; built on demand by Task 8's tests via
14//!   `cargo build --manifest-path …`). Statically-linked test binary that
15//!   performs predictable I/O so landlock V1 path resolution doesn't trip
16//!   on dynamic linker probing or `/bin → /usr/bin` symlinks.
17//! - `fixtures/projects/<plugin>/` — per-plugin tau project fixture
18//!   (Task 5). Each contains a minimal `tau.toml` + `.tau/config.toml`
19//!   declaring `[sandbox] required_tier = "strict"` plus any cassette /
20//!   data files needed.
21//! - `tests/layer3_check_sandbox.rs` (Task 6) — 5 tests; install plugin
22//!   into tempdir, run `tau resolve --check-sandbox`, assert exit 0.
23//! - `tests/layer4_container.rs` (Task 7) — 5 tests; force Container
24//!   adapter, drive each plugin's golden path.
25//! - `tests/layer4_native.rs` (Task 8) — 5 tests; force Native adapter,
26//!   exercise Task 3's symlink-resolution fix. Linux-only.
27//!
