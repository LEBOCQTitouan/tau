Module resolver

Declarative-requirements adapter resolver.

resolve_adapter implements the Bazel-style filter pipeline from spec §3:

  1. Platform filter — only consider adapters that apply to the current OS.
  2. Probe filter — instantiate the adapter and call probe(). Adapters returning tau_ports::SandboxProbe::Unavailable are rejected.
  3. Tier filter — the adapter’s delivered tier must be ≥ the effective required tier (max of project tier and plugin-floor tier).
  4. Shape filter — every shape in requirements.required_shapes must be in the adapter’s supported_shapes.
  5. Per-plugin tier filter — every plugin’s required_tier must be ≤ the adapter’s delivered tier.
  6. Priority sort — among survivors, pick the one with the highest AdapterRegistration::priority.

Mock injection via TAU_TESTING_ALLOW_MOCK_SANDBOX=1 bypasses the registry entirely; it is an opt-in, never a silent fallback.
