Module resolve 

Source
Expand description

Resolve transitive requires.tools dependencies for a project’s agents into a concrete install plan.

Three phases (see spec §5):

  1. Group input entries by name.
  2. Conflict checks per name: source equality + version-constraint compatibility.
  3. Pick a concrete Version per name: lockfile reuse if installed and compatible, else list available versions at source and pick the highest satisfying all constraints.

One level deep at v0.1: we resolve agents’ requires.tools only; recursive package-level dependencies resolution stays deferred per ADR-0004 §10.
