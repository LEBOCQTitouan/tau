Expand description

Errors from install and install_with_options (added in Task 10).

Composes GitError, ManifestReadError, RegistryError, and ScopeError via #[from] so the install lifecycle can use ? propagation throughout.

Variants (Non-exhaustive)§

This enum is marked as non-exhaustive
Non-exhaustive enums could have additional variants added in future. Therefore, when matching against variants of non-exhaustive enums, an extra wildcard arm must be added to account for any future variants.
§

Git(GitError)

The git subprocess failed.

§

Manifest(ManifestReadError)

Manifest parsing or structural validation failed.

§

Registry(RegistryError)

Lockfile read / write failed.

§

Scope(ScopeError)

Scope resolution / management failed.

§

SourceManifestMismatch

User-supplied source doesn’t match the manifest’s declared source.

Fields

§expected: String

The source the user passed to install, formatted via Display.

§found: String

The source declared in the cloned package’s tau.toml, formatted via Display.

§

Locked

Another install operation already holds the per-scope advisory lock.

Fields

§scope: String

Lossy-UTF-8 path of the scope state directory.

§

BuildFailed

cargo build exited non-zero while building a kind = "rust-cargo" plugin. Carries the cargo exit status and the last ~4 KiB of cargo’s stderr so users can diagnose the compile failure without re-running. The cloned source is left on disk; users retry via tau install --force or inspect the staging area.

Fields

§exit_status: ExitStatus

Exit status returned by the cargo build subprocess.

§stderr_tail: String

Last ~4 KiB of cargo’s stderr (lossy UTF-8) for diagnostic display.

§

CargoNotFound

The cargo binary was not found at the configured location. Either BuildOptions::cargo_path was set to a non-existent path, or the default discovery (cargo on PATH) found nothing.

§

CrossCheck

Layer 2 cross-check detected a mismatch between the plugin binary’s claimed capabilities and those declared in the manifest.

The installed binary remains on disk. Users should fix their manifest and retry via tau install --force.

Fields

§message: String

Human-readable description of the cross-check failure.

§

SkillContentMissing

Skills-2: kind = "skill" package’s SKILL.md is absent at install time.

Path comes from manifest.skill().unwrap().content (default "SKILL.md") joined with the install directory.

Fields

§name: String

Skill package name (for human-readable error).

§expected_path: PathBuf

Absolute path the install pipeline tried to read.

§

SkillNameMismatch

Skills-2: SKILL.md frontmatter’s name field does not match the package’s tau.toml name. Both must equal.

Fields

§tau_toml: String

name field from tau.toml.

§skill_md: String

name field from SKILL.md frontmatter.

§

SkillFrontmatterInvalid

Skills-2: SKILL.md frontmatter failed to parse, or is missing the required name / description fields.

Fields

§detail: String

Human-readable reason (e.g. “missing required field name”, “YAML parse error: …”).

§

SkillReferenceWithoutCapability

Skills-2: SKILL.md body contains ${SKILL_DIR}/<rel-path> references but no [[capabilities]] kind = "fs.read" glob covers the path. Hard-fail because runtime would fail with a confusing capability-denied error mid-task; install time is the right gate.

Fields

§reference: String

The offending ${SKILL_DIR}/<path> reference found in the body.

§declared_paths: Vec<String>

fs.read glob entries declared in the manifest at install time (so the user can see what to extend).

§

NotASkillPackage

Skills-5: cloned source has neither tau.toml nor SKILL.md. Not a recognized skill package format.

Fields

§path: PathBuf

The workspace directory that was inspected.

§detail: String

Human-readable detail.

§

SynthesizeFailed(SynthesizeError)

Skills-5: detected Anthropic format but SKILL.md synthesis failed.

§

Internal

Catch-all for install lifecycle failures not yet covered by typed variants. Use this only when the failure cannot be reported as Git, Manifest, Registry, Scope, SourceManifestMismatch, or Locked. See: escape-hatches.md#installerror-internal.

Fields

§message: String

Human-readable message describing the internal failure.
