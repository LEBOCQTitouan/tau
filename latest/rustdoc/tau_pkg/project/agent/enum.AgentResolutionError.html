Expand description

Errors from build_agent_definition.

Per spec §3.4. All variants carry the offending agent_id so the CLI can emit “agent X: …” style messages without further plumbing.

Variants (Non-exhaustive)§

This enum is marked as non-exhaustive
Non-exhaustive enums could have additional variants added in future. Therefore, when matching against variants of non-exhaustive enums, an extra wildcard arm must be added to account for any future variants.
§

PackageNotFound

The agent’s package reference doesn’t match any installed package by name in the resolved scope.

Fields

§agent_id: String

Agent id from [agents.<id>].

§package: String

Raw package = "..." reference from the entry.

§

PackageVersionUnsatisfied

Installed packages with the requested name exist but none satisfy the version requirement.

Fields

§agent_id: String

Agent id from [agents.<id>].

§package: String

Raw package = "..." reference from the entry.

§req: String

Stringified semver requirement.

§

LlmBackendNotFound

The agent’s llm_backend reference doesn’t match any installed package in the resolved scope. v0.1 does NOT enforce that the package’s kind is llm-backend — that check lands when the runtime grows kind-aware loading (Phase 1+).

Fields

§agent_id: String

Agent id from [agents.<id>].

§backend: String

Raw llm_backend = "..." from the entry.

§

§agent_id: String

PackageParse

The package = "name@^0.1" string failed to parse into name + semver requirement.

Fields

§agent_id: String

Agent id from [agents.<id>].

§package: String

Raw package = "..." reference.

§message: String

Human-readable parse failure message.

§

Registry

Failed to call into tau-pkg’s registry layer (filesystem error, schema mismatch, etc.).

Fields

§agent_id: String

Agent id from [agents.<id>].

§source: RegistryError

Underlying tau-pkg registry error.

§

InvalidIdentifier

Failed to construct an AgentId or PackageName from a string the project config supplied. Usually means the ASCII kebab-case invariant is violated.

Fields

§agent_id: String

Agent id from [agents.<id>] (may itself be the invalid id).

§message: String

Human-readable message describing what was invalid.
