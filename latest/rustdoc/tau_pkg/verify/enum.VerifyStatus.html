        expected: String,
        actual: String,
    },

    },

    },

    SkillContentDrift {
        name: String,
        expected: String,

    },

        skill_name: String,
        issue: AnthropicConformanceIssue,
    },
}
Expand description

Per-package verification status.

Variants (Non-exhaustive)§

This enum is marked as non-exhaustive
Non-exhaustive enums could have additional variants added in future. Therefore, when matching against variants of non-exhaustive enums, an extra wildcard arm must be added to account for any future variants.
§

Ok

Hash matches the lockfile entry. No drift.

§

TreeDrift

Source tree SHA-256 doesn’t match LockedVersion.sha256.

Fields

§expected: String

Hex hash from the lockfile.

§actual: String

Hex hash recomputed now.

§

BinaryDrift

Plugin binary SHA-256 doesn’t match LockedPlugin.binary_sha256.

Fields

§path: PathBuf

Path to the binary.

§expected: String

Hex hash from the lockfile.

§actual: String

Hex hash recomputed now.

§

Missing

Install dir doesn’t exist on disk.

Fields

§path: PathBuf

The install dir the lockfile pointed to.

§

Unverified

Lockfile entry has empty sha256 (v2-leftover). Informational; not drift.

§

SkillContentDrift

A skill package’s SKILL.md content hash differs from the install-time snapshot recorded in the lockfile. Parallel to BinaryDrift for plugin binaries.

Remediation: re-run tau install <skill> to refresh.

Fields

§name: String

Package name.

§expected: String

Expected SHA-256 (hex; from the lockfile).

§got: String

Actual SHA-256 (hex; from re-hashing the on-disk file).

§

AnthropicConformance

Skills-5: skill failed Anthropic-format conformance check (triggered by the --anthropic-strict flag on tau verify).

Fields

§skill_name: String

Name of the affected skill package.

§issue: AnthropicConformanceIssue

Specific conformance issue detected.
