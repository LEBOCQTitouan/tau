pub fn build_agent_definition(

Expand description

Resolve a project [agents.<id>] entry to the kernel-ready (AgentDefinition, PackageManifest) pair.

Per spec §3.3 steps 1–6:

  1. Parse entry.package into (name, semver req).
  2. List installed packages in scope; pick the highest version of name satisfying the requirement.
  3. Read the resolved package’s manifest from scope.package_dir(name, version)/tau.toml.
  4. Verify entry.llm_backend resolves to an installed package.
  5. (Removed) requires.tools is no longer verified here; resolve+install happens at the CLI call sites before build_agent_definition is called.
  6. Read prompt.system / prompt.system_file (path relative to project_root) into Option<String>.
  7. Construct AgentDefinition::new(...) chained with .with_system_prompt(...) and .with_config(...).

Returns the manifest alongside the definition because the kernel needs the manifest’s capability declarations and dependencies for Runtime::run (per the tau-runtime amendments in Tasks 2 & 3).
